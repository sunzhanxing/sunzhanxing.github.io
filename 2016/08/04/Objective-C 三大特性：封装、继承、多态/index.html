Objective-C 三大特性：封装、继承、多态

发表于 2016-08-04   |     |   阅读次数

初学面向对象程序设计中的类有三大特性：继承，封装，多态，这个也是介绍类的时候，必须掌握的前提，所以今天来讲解下Objective－C中类的三大特性：封装、继承、多态

封装

封装：隐藏内部（定义类、定义属性、定义方法。）实现，稳定外部接口。

继承中对问题的分析：

1.在继承中，子类对象如何正确发送方法的机制

每一个Objective－C对象都有一个隐藏的指针（isa）指向类的代码，当对象发送消息的时，当前的对象会首先在当前类里去查找对应的方法，如果找到的话，直接执行代码；如没有找到，则去父类里找对应的方法，如果找到的话，就会执行。如果找不到继续按照这样的原则，沿着继承链向上查找，直到根类NSObject（这个类是Objective - C所有类的根类
），如果在这个NSObject类中仍找不到相应的方法，那么程序直接崩溃掉。

2.如何在子类里面，直接调用父类方法或者父类的实例变量。（后续会特意开一章介绍Super与self）

采用Super关键字，需要注意的是，Super 是一个关键字，当你使用 Super 的时候，编译器知道，你现在的目的只是访问父类的一些元素。（这里的 Super 与 self 并不是一个概念：super 是一个关键字，而 self 只是一种调用机制，并不是关键字）

多态

多态：不同对象对同一消息的不同响应，子类可以重写父类的方法（父类指针指向子类对象）

重写的概念:重写的概念，只存在于继承语法中。是指：父类声明一个方法，但是父类并没有真正意义的去实现这个方法（或者说实现了，向按照自己的想法再去实现下），而是在子类中真正的实现方法的功能，这种形式就叫做对这个方法的重写。（重写方法的目的是为了实现面向对象编程里的多态）。

例如：假设车类Car都用有一个相同的 方法 -(void)print; ,那 Bus 属于车类，Bike 也属于车类，都继承了 Car 类后，实现各自的 print ，但是调用时我们只需调用各自的 print 方法。也就是不同的对象以自己的方式响应了相同的消息（响应了 print 这个方法）。
